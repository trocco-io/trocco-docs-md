## 概要

Yahoo!広告 ディスプレイ広告 オーディエンスリストへデータを転送する設定のDocsです。

## 制約

- データ数が100件未満の場合、Yahoo!広告 ディスプレイ広告の仕様によりエラーとなります。

## 設定項目

### STEP1 基本設定

| 項目名 | 必須 | デフォルト値 | 内容 |
| --- | --- | --- | --- |
| Yahoo!広告 API接続情報 | Yes | - | あらかじめ登録してあるYahoo!広告 API接続情報を選択します。 |
| ベースアカウントID | Yes | - | 接続情報のYahoo! JAPANビジネスIDが直接権限を持つアカウント（MCCアカウント・広告アカウント）を選択します。<br>選択したアカウント、およびその配下全てのアカウントに対して操作が可能になります。|
| アカウントID | Yes | - | 選択したベースアカウントIDで操作可能な、データ取得対象のアカウントIDを入力します。<br>アカウントIDは、ディスプレイ広告の管理画面から確認できます。 |
| オーディエンスリスト名 | Yes | - | オーディエンスリスト名を入力します。<br>存在しないオーディエンスリスト名を入力した場合は、新しくオーディエンスリストが作成されます。 |
| オーディエンスリストの説明 | No | - | オーディエンスリストの説明を入力します。 |
| 連携データタイプ | Yes | - | 転送する顧客データの種類を選択します。<br>詳しくは、後述の[連携データタイプについて](/docs/data-destination-yahoo-display-ads-audience-list#連携データタイプについて)を参照ください。|
| 転送モード | Yes | - | 転送モードを選択します。<br>詳しくは、後述の[転送モードについて](/docs/data-destination-yahoo-display-ads-audience-list#転送モードについて)を参照ください。|
| データのアップロード完了まで待つ | No | 無効 | 転送先による転送データの確認が完了するまで、転送ジョブを継続するかを選択します。<br>詳しくは、後述の[転送ジョブの継続設定について](/docs/data-destination-yahoo-display-ads-audience-list#転送ジョブの継続設定について)を参照ください。 |

#### 連携データタイプについて

| 連携データタイプ | 内容 |
| --- | --- |
| メールアドレス・電話番号 | メールアドレスまたは電話番号、もしくは両方を転送対象とします。<br>SHA-256でハッシュ化し送信する必要があります。|
| Android端末用広告識別子 | Android端末用の広告識別子（AAID）を転送対象とします。|
| iOS端末用広告識別子 | iOS端末用の広告識別子（IDFA）を転送対象とします。|

:::(Warning) (端末用広告識別子の転送について)
オプトアウトした顧客の端末用広告識別子は転送しないでください。

:::

#### 転送モードについて

| 転送モード | 内容 |
| --- | --- |
| 追加 | 既存のオーディエンスリストに顧客データが追加されます。<br>オーディエンスリストが存在しない場合は、オーディエンスリストが作成されます。 |
| 洗い替え | 転送設定を新規作成する際は選択できません。<br>リストが削除され、同じオーディエンスリスト名で再作成されます。<br>オーディエンスリストが存在しない場合は、オーディエンスリストが作成されます。 |

:::(Warning) (転送モードの「洗い替え」について)

転送モードの「洗い替え」については、2026年8月1日（土）にサポート終了を予定しています。
詳しくは、[転送先Yahoo!広告 ディスプレイ広告オーディエンスリスト「洗い替え」モード廃止のお知らせ（2026年8月1日予定）](https://help.trocco.io/hc/ja/articles/5440262254110)を参照ください。

### STEP1 詳細設定

| 項目名 | 必須 | デフォルト値 | 内容 |
| --- | --- | --- | --- |
| データの有効期間（日） | Yes | - | オーディエンスリストに連携したデータの有効期間を設定します。1〜60日の間で設定できます。|

#### 転送ジョブの継続設定について

- **「有効」を選択した場合**
  - Yahoo!広告側で転送データの確認が完了するまで、転送ジョブを継続します。
  - 有効にすると、アップロード処理の完了を確認できますが、最大48時間程度かかることがあります。詳細は[公式ドキュメント](https://ads-help.yahoo-net.jp/s/article/H000044507)をご確認ください。
- **「無効」を選択した場合**
  - Yahoo!広告側で転送データの確認が完了するのを待たずに、転送ジョブを終了します。
  - 転送データの確認完了を待たずに転送ジョブが終了するため、転送時間を節約できます。
  - Yahoo!広告側の転送データの確認ステータスを確認する方法は、[データのアップロード状況を確認する](https://ads-help.yahoo-net.jp/s/article/H000044507?language=ja#c07)を参照ください。

## カラムマッピング

Yahoo!広告 ディスプレイ広告 オーディエンスリストにデータを転送するためには、TROCCO上でカラム名や型をマッピングする必要があります。
以下の表を参照し、TROCCOの転送設定STEP2「カラム定義」タブにて、マッピングを行ってください。
詳しくは[公式ドキュメント](https://ads-help.yahoo-net.jp/s/article/H000044507?language=ja#c01)を参照ください。

* STEP1 基本設定で選択した「連携データタイプ」によって、必要なマッピングが異なります。指定したデータタイプ以外のカラムは定義しないでください。
* 必須列に`(*)`が付いたカラム（パラメータ）は、転送対象のデータに**少なくとも1つ**含めてください。

### 連携データタイプに「メールアドレス・電話番号」を選択した場合

| カラム名 | データ型 | 必須 | 備考 |
| - | - | - | - |
| `email` |`string`| Yes(\*) | メールアドレス。<br>半角小文字英数字に変換した上で、SHA-256でハッシュ化し送信する必要があります。 |
| `phone_number` |`string`| Yes(\*) | 国番号付きの電話番号。<br>ハイフン無し半角数字に変換した上で、SHA-256でハッシュ化し送信する必要があります。 |

### 連携データタイプに「Android端末用広告識別子」を選択した場合

| カラム名 | データ型 | 必須 | 備考 |
| - | - | - | - |
| `aaid` |`string`| Yes | Android端末用広告識別子。 |

### 連携データタイプに「iOS端末用広告識別子」を選択した場合

| カラム名 | データ型 | 必須 | 備考 |
| - | - | - | - |
| `idfa` |`string`| Yes | iOS端末用広告識別子。 |
