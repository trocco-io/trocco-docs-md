## 概要

処理時間画面では、以下の機能を利用できます。

- **処理時間の内訳の確認**：転送設定やデータマート定義・ワークフローなど、定義ごとの処理時間を確認できます。
- **リソースグループ別の処理時間の確認**：リソースグループおよび各ジョブの種類ごとに、各ジョブの処理時間や合計処理時間を確認できます。処理時間が多いリソースグループを特定して調整するなど、TROCCO の運用面における効率化・最適化にお役立てください。
- **処理時間通知の設定**：月のご契約の処理時間に対する使用率が指定した閾値を超過した際に、任意の通知先へ通知を送信できます。ご契約の処理時間超過を検知し、運用の見直しやリソース配分の調整に役立てることができます。

## 処理時間分析

転送設定やデータマート定義・ワークフローなどの処理時間についての集計結果を過去1年分確認できます。
月末の処理時間超過原因の特定や定義の棚卸し・最適化などに活用できます。

### 機能

**処理時間サマリー**で全体の集計結果を、**処理時間エクスプローラー**で各定義ごとの詳細情報を確認できます。

:::(Info) (サマリーとエクスプローラーの不一致について)

処理時間サマリーとエクスプローラーの数値について、当日分のデータにズレが発生することがあります。
:::

#### 処理時間サマリー

全体の処理時間についてのサマリーを確認できます。

- **累積処理時間**
- **月末着地予想**
- **直近6ヶ月＋来月予測（当月ベース）**

#### 処理時間エクスプローラー

転送設定やデータマート定義・ワークフローなどの定義ごとに、処理時間や実行回数などを確認できます。
期間や集計に含める設定タイプの指定が可能です。

:::(Info) (閲覧権限のない定義の表示)

閲覧権限のないユーザーに対しては、エクスプローラーの各定義は「閲覧権限なし」と表示されますが、処理時間や実行回数などは確認できます。

:::

## リソースグループ別の処理時間

[チーム機能](/docs/about-team)をご利用の場合、処理時間画面の**リソースグループ別の処理時間**タブから、リソースグループごとの処理時間の内訳を確認できます。
なお、リソースグループに紐づけられていないジョブについては、「リソースグループに属さないリソース」として集計し表示されます。

:::(info) (リソースグループ別の処理時間のご利用について)
リソースグループ別の処理時間を利用できるのは、チーム機能オプションを有効にしているお客様に限ります。
:::

:::(Info) (処理時間の計算に使われるタイムゾーンについて)
タイムゾーンについては、お客様の設定のタイムゾーン設定にかかわらず日本時間にて集計されます
:::

:::(Info) (削除済みのリソースグループについて)
ジョブ実行時点でジョブ定義に紐づけられていたリソースグループがその後削除された場合、本画面においては「削除済みリソースグループ」として集計し表示されます
:::

:::(Warning) (処理時間機能のデータについて)
処理時間機能に反映されるデータは 2024 年 11 月初日以降のデータのみです。
:::

### 各ラベルの説明

| 列名                       | 説明                                                                                                                             |
| -------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| リソースグループ名         | 集計対象となるリソースグループ名です                                                                                             |
| 合計処理時間               | 集計対象となるリソースグループに紐づくジョブ実行により消費された処理時間です                                                     |
| 合計処理時間比率           | 合計処理時間を月のご契約の処理時間で割った比率です                                                                               |
| データ転送処理時間         | 該当リソースグループに紐づくデータ転送ジョブの処理時間のうち、リソース増強オプションを設定していない転送ジョブの処理時間合計です |
| データ転送処理時間(Large)  | 該当リソースグループに紐づくデータ転送ジョブの処理時間のうち、リソースサイズを `Large` に設定した転送ジョブの処理時間です (\*1)  |
| データ転送処理時間(XLarge) | 該当リソースグループに紐づくデータ転送ジョブの処理時間のうち、リソースサイズを `XLarge` に設定した転送ジョブの処理時間です (\*1) |
| データマート処理時間       | 該当リソースグループに紐づくデータマートジョブの処理時間合計です                                                                 |
| dbt 処理時間               | 該当リソースグループに紐づく dbt 連携ジョブの処理時間合計です                                                                    |
| ワークフロータスク処理時間 | 該当リソースグループに紐づくワークフロータスクの処理時間合計です (\*2)                                                           |

\*1 プロフェッショナルプランをご契約いただいていないお客様の場合、固定で `00m00s` の値が表示されます。
\*2 転送ジョブ、データマートジョブ、dbt 連携ジョブは、ワークフロー経由から実行された場合であってもそれぞれのジョブの集計対象として計算されます。

## 処理時間通知

処理時間の使用率が指定した閾値を超過した際に、任意の通知先へ通知を送信する設定ができます。
通知先の作成方法については、[通知設定について](/docs/notification)を参照してください。

:::(Warning) (通知のタイミングについて)

処理時間通知はリアルタイムではなく、定期的に送信されます。
閾値の超過から通知の送信まで、数時間程度の遅延が生じることがあります。

:::

:::(Info) (特権管理者向けメール通知について)

処理時間通知設定の有無にかかわらず、特権管理者には、月次の処理時間が50% / 80% / 100% を超過した際に、システム通知メールが自動で送信されます。
特権管理者以外でも処理時間の動向を把握したい場合や、メール以外の通知手段が必要な場合などには、処理時間通知をご利用ください。

:::

### 操作権限

通知設定を操作できるロールは以下のとおりです。

- アカウント特権管理者：すべての通知設定の作成・編集・削除・閲覧
- アカウント管理者：自身が作成した通知設定の作成・編集・削除・閲覧
- アカウントメンバー：閲覧・操作不可
